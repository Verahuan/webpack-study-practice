【知识点】 什么是webpack? 前端模块化打包工具 现代js应用的静态模块打包工具 sass、less等等，浏览器不支持，需要编译成css es6编译成es5 打包工具：grunt/gulp/webpack/rollup(vue源码构建方法) 【补】什么是前端模块化？ 模块化规范：AMD、CMD、CommonJS、ES6(!浏览器支持) webpack也可以把不同的前端模块化规范转换为浏览器可以识别的模块化规范 webpack处理模块之间的依赖关系（js、css等都当做模块） TS->JS 与grunt和gulp的对比？ grunt和gulp的核心是task（称作：自动化任务管理工具） 我们配置对应的事务，其一次来执行这些任务（ES6、TS转换、图片压缩、sass等等） 如果是模块化的话，用webpack比较好 webpack本身依赖于node环境 npm:node packges manager (npm):node为了正常执行很多代码，必须包含各种依赖的包，这些包由npm工具进行管理 文件夹 --src --dist(distribution) webpack的配置 准备： path路径 node语法 使用node语法前，需要进行初始化 npm init 生成package.json （如果需要单独依赖node环境，一般都需要这个文件） npm install 根据package.json安装一些依赖 在webpack.config.js文件中进行配置 const path=require("path")//node包里面的path包 module.export={ entry: output:{
